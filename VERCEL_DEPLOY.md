# دیپلوی X4G روی Vercel (نسخه‌ی تطبیق‌یافته)

## ⚠️ قبل از شروع بخوانید
این نسخه برای اجرا روی Vercel Functions (با Fluid Compute) تطبیق داده شده، اما دو محدودیت اساسی همچنان وجود دارد:

1. **قطع دوره‌ای تونل:** هر اتصال WebSocket حداکثر ۸۰۰ ثانیه (~۱۳ دقیقه) روی پلن Pro زنده می‌ماند (تا ۳۰ دقیقه در حالت بتا با تنظیم دستی). بعد از آن کلاینت VLESS باید خودش reconnect کند. اکثر کلاینت‌ها (v2rayNG، NekoBox، Streisand) این کار را خودکار انجام می‌دهند، اما یعنی گاهی چند ثانیه قطعی خواهید داشت.
2. **بدون Redis، هیچ داده‌ای ذخیره نمی‌شود:** لینک‌ها، گروه‌های ساب و رمز پنل باید در Redis نگه داشته شوند چون فایل‌سیستم Vercel موقتی است.

اگر تونل پایدار و بدون قطعی می‌خواهید، Railway/Render/Fly.io/VPS همچنان انتخاب بهتری هستند.

---

## ۱) ساخت دیتابیس Redis
1. در داشبورد Vercel پروژه را باز کنید → تب **Storage** → **Marketplace Database** یا **Create Database**.
2. یک دیتابیس **Redis** (مثلاً Upstash) بسازید و به پروژه Connect کنید.
3. Vercel به‌صورت خودکار متغیر محیطی `REDIS_URL` (یا `KV_URL`) را به پروژه اضافه می‌کند. اگر نام متغیر چیز دیگری بود (مثلاً `KV_REST_API_URL` که آدرس REST است، نه پروتکل redis://)، حتماً مقدار `redis://...` واقعی را پیدا کرده و به‌صورت دستی در Environment Variables با نام `REDIS_URL` ست کنید.

## ۲) متغیرهای محیطی مورد نیاز
در تنظیمات پروژه → Settings → Environment Variables این‌ها را اضافه کنید:

| متغیر | مقدار |
|---|---|
| `REDIS_URL` | آدرس Redis (مرحله ۱) |
| `ADMIN_PASSWORD` | رمز عبور پنل مدیریت (پیش‌فرض کد: 2010 — حتماً عوضش کنید) |
| `SECRET_KEY` | یک رشته‌ی تصادفی طولانی و ثابت (اگر ندهید هر دیپلوی جدید یک مقدار تصادفی جدید تولید می‌کند و لینک پیش‌فرض عوض می‌شود) |

## ۳) دیپلوی
1. این پوشه را در یک ریپازیتوری گیت‌هاب push کنید (یا مستقیماً با `vercel` CLI از همین پوشه دیپلوی کنید: `vercel --prod`).
2. در Vercel → **Add New Project** → ریپو را انتخاب کنید. Framework Preset را می‌توانید روی **Other** بگذارید؛ Vercel به‌خاطر `requirements.txt` و `main.py` به‌صورت خودکار آن را به‌عنوان اپ FastAPI تشخیص می‌دهد.
3. Deploy بزنید.

## ۴) تست
- `https://your-project.vercel.app/health` باید `{"status":"ok",...}` برگرداند.
- `https://your-project.vercel.app/dashboard` باید صفحه‌ی لاگین/داشبورد را نشان دهد (با `ADMIN_PASSWORD` وارد شوید).
- یک کانفیگ VLESS بسازید و در کلاینت تست کنید؛ اگر هر ۱۳-۳۰ دقیقه یک قطعی کوتاه دیدید، طبیعی و ناشی از محدودیت Vercel Functions است.

## نکات فنی تغییرات اعمال‌شده نسبت به نسخه‌ی اصلی
- `load_state` / `save_state`: به‌جای نوشتن در فایل `/data/x4g_state.json`، حالا در Redis (کلید `x4g:state`) ذخیره می‌شوند.
- سشن‌های لاگین پنل هم در Redis نگه داشته می‌شوند (نه فقط در حافظه)، چون هر request روی Vercel ممکن است به یک نمونه‌ی متفاوت برسد.
- تشخیص دامنه (`get_host`) حالا علاوه بر `RAILWAY_PUBLIC_DOMAIN`، از `VERCEL_PROJECT_PRODUCTION_URL` / `VERCEL_URL` هم پشتیبانی می‌کند.
- آمار زنده (تعداد اتصالات لحظه‌ای، لاگ فعالیت، ترافیک ساعتی) همچنان فقط در حافظه‌ی هر نمونه است و بین نمونه‌های مختلف Vercel یکسان نخواهد بود — این‌ها صرفاً نمایشی هستند و مشکلی برای عملکرد تونل ایجاد نمی‌کنند.
- `vercel.json` اضافه شده تا `maxDuration` روی ۸۰۰ ثانیه (حداکثر پایدار پلن Pro) تنظیم شود.
